# 物联网安全论文综述

本综述按年份排序，对5篇重要物联网安全相关论文进行分析，提取其标题、作者、年份、期刊、摘要、研究内容、技术方案、解决的问题及创新点。

---

## 1. 物联网安全综述（2017年）

**论文标题：** 物联网安全综述  
**作者：** 张玉清, 周威, 彭安妮  
**年份：** 2017  
**期刊：** 计算机研究与发展  
**卷期：** 54(10): 2130-2143  

**摘要：**
随着智能家居、数字医疗、车联网等技术的发展，物联网应用越发普及，其安全和隐私也受到越来越多研究者的关注。目前，物联网安全的相关研究尚在起步阶段，大部分研究成果还不能完善地解决物联网发展中的安全问题。本文首先对物联网3层逻辑架构进行了介绍，阐述了每个层次的安全问题与研究现状重点；然后分析并讨论了物联网的主要应用场景（智能家居、智能医疗、车联网、智能电网、工业与公共基础设施）中需要特别关注的隐私保护、入侵检测等安全问题；再次，归纳分析了现有研究工作中的不足与安全问题产生的主要原因，指出物联网安全存在的五大技术挑战：数据共享的隐私保护方法、有限资源的设备安全保护方法、更加有效的入侵检测防御系统与设备测试方法、针对自动化操作的访问控制策略、移动设备的跨域认证方法；最后，通过详尽分析这五大技术挑战，指出了物联网安全未来的研究热点。

**研究内容：**
1. 物联网架构安全性分析（感知层、传输层、应用层）
2. 物联网应用场景安全分析（智能家居、智能医疗、车联网、智能电网、工业与公共基础设施）
3. 现有研究不足与安全问题成因分析
4. 五大技术挑战的提出与分析
5. 未来研究方向展望

**技术方案：**
1. 三层逻辑架构安全分析方法
2. 轻量级密码学算法（针对感知层设备资源受限问题）
3. 基于物理不可克隆函数（PUF）的认证和密钥生成协议
4. 基于生物特征（步态、滑动屏幕力度）的设备认证
5. 同态加密技术（用于数据共享时的隐私保护）
6. 区块链技术（实现物联网设备匿名共享）

**解决的问题：**
1. 物联网设备资源有限，难以部署传统安全措施
2. 物联网设备分布广泛，物理安全难以保障
3. 数据共享过程中的隐私泄露问题
4. 缺乏有效的入侵检测和防御系统
5. 自动化操作的访问控制问题
6. 移动设备的跨域认证问题

**创新点：**
1. 系统性归纳了物联网安全面临的核心挑战，并提出五大技术挑战框架
2. 强调物联网安全研究与应用场景、架构层次的关联性
3. 从威胁、检测、防御三个维度系统性地总结了研究现状
4. 基于设备自身物理特性的认证方法（PUF）
5. 结合用户生物特征的双因素认证方案

## 2. 物联网安全研究综述：威胁、检测与防御（2021年）

**论文标题：** 物联网安全研究综述：威胁、检测与防御  
**作者：** 杨毅宇, 周威, 赵尚儒, 刘聪, 张宇辉, 王鹤, 王文杰, 张玉清  
**年份：** 2021  
**期刊：** 通信学报  
**卷期：** 42(8): 189-205  

**摘要：**
基于近5年网安国际会议（ACM CCS、USENIX Security、NDSS、IEEE S&P）中发表的物联网安全文献，以及其他部分高水平研究工作，从威胁、检测、防御的视角对物联网安全研究工作进行了系统的整理和分析。首先，介绍了物联网系统的基本架构。然后，将当前研究中提出的主要威胁分为8种类型，并分析了威胁的成因和危害。之后，介绍了针对这些威胁所提出的6种威胁检测和5种防御方案，并对比了它们的技术原理和优缺点。最后，提出了当前研究依然面临的主要挑战，并指出了未来研究发展的方向。

**研究内容：**
1. 物联网系统架构分析（感知层、网络层、应用层）
2. 8类安全威胁分类与分析（云平台访问控制缺陷、云平台恶意应用、云平台实体和应用交互漏洞、通信协议漏洞、通信流量侧信道信息泄露、设备固件漏洞、基于语音信道的攻击、基于物联网设备的僵尸网络）
3. 6类威胁检测技术（云平台恶意应用检测、云平台实体和应用交互漏洞检测、基于静态分析的固件漏洞检测、基于动态分析的固件漏洞检测、基于手机App的固件漏洞检测、基于侧信道的设备异常检测）
4. 5类防御方案（细粒度的云平台访问控制、安全的通信协议、流量特征隐藏、基于可信计算的固件安全防护机制、语音攻击防御）

**技术方案：**
1. 数据流追踪技术（用于云平台恶意应用检测）
2. 黑盒语音测试技术（针对语音平台）
3. 符号执行和污点分析技术（用于静态固件分析）
4. 二进制相似性检测技术（基于计算机视觉的特征提取）
5. QEMU仿真环境构建
6. App-based模糊测试（利用手机App作为设备输入接口）
7. 侧信道特征提取（Wi-Fi信号强度、包长度、时间戳等）
8. 细粒度访问控制模型（ContextIoT、SmartAuth）
9. 安全通信协议设计（安全配对协议）
10. 流量封装和塑形技术（隐藏流量特征）
11. 可信计算技术（控制流完整性保护、远程认证）

**解决的问题：**
1. 云平台权限管理粒度过粗问题
2. 云平台间授权漏洞
3. 云平台应用间执行冲突问题
4. 通信协议缺乏内建安全机制
5. 通信流量侧信道隐私泄露
6. 固件漏洞难以检测的问题
7. 语音信道隐藏攻击问题
8. 物联网设备僵尸网络威胁

**创新点：**
1. 首次系统性地从"威胁、检测、防御"三个维度综述物联网安全
2. 提出基于上下文信息的细粒度访问控制方案
3. 结合计算机视觉技术的二进制相似性检测
4. 基于手机App的固件漏洞检测方法
5. 多种侧信道特征的综合利用方案

## 3. 物联网固件漏洞安全检测综述（2022年）

**论文标题：** 物联网固件漏洞安全检测综述  
**作者：** 李涛, 田迎军, 葛阳晨, 田源, 李剑  
**年份：** 2022  
**期刊：** 信息安全研究  
**卷期：** 8(12): 1146-1155  

**摘要：**
随着万物互联时代的到来，物联网的安全问题越来越突出，尤其是物联网中由于固件漏洞引起的安全隐患或攻击行为导致的经济损失越来越大。高效的固件漏洞检测技术越来越成为保障物联网设备安全的关键，因此研究物联网中固件漏洞安全检测相关方法与技术具有重要的理论意义和实用价值。分析了物联网固件安全问题频发的原因，归纳了物联网固件面临的主要安全威胁，针对固件漏洞分析所面临的挑战，综述了现有固件漏洞检测方法。通过对不同方法优势与不足的分析，为进一步提升固件安全缺陷检测方法的智能化、精准化、自动化、有效性、可扩展性提供指导。最后，对物联网固件漏洞安全检测进行了展望。

**研究内容：**
1. 物联网固件安全问题频发的原因分析（开发人员安全意识不足、厂商缺乏安全意识、固件设备计算存储资源有限）
2. 物联网固件面临的安全威胁分类（固件认证绕过漏洞、内存漏洞、逻辑漏洞、命令注入漏洞、固件修改漏洞）
3. 固件漏洞分析面临的挑战（固件难以获取、固件格式差异较大、符号信息缺失、仿真困难、运行环境受限）
4. 固件漏洞检测方法综述（静态分析、符号执行、模糊测试、机器学习）

**技术方案：**
1. 固件获取方法（通过爬虫获取、模拟设备行为诱骗下载、App内提取、通过硬件提取）
2. 静态分析技术（类型分析、格理论、数据流与控制流分析、程序切片、抽象解释）
3. 符号执行技术（KLEE、SAGE等）
4. 模糊测试技术（黑盒模糊测试、灰盒模糊测试）
5. 机器学习技术（深度学习算法挖掘已知漏洞特征）

**解决的问题：**
1. 固件源码获取困难
2. 固件代码复杂多样，反编译困难
3. 设备资源受限，难以部署传统防御措施
4. 固件分析工具的可扩展性差
5. 固件代码保护机制带来的分析阻碍

**创新点：**
1. 系统性分析了物联网固件安全问题频发的根本原因
2. 针对固件分析的四大挑战（获取、解包、符号信息、仿真、运行环境）进行了全面梳理
3. 提出将机器学习技术应用于固件漏洞检测的思路
4. 强调静态分析、符号执行、模糊测试、机器学习四类技术的互补关系
5. 指出未来应关注智能化、精准化、自动化、有效性、可扩展性五个方向

## 4. 物联网设备固件自动化漏洞挖掘技术研究综述（2025年）

**论文标题：** 物联网设备固件自动化漏洞挖掘技术研究综述  
**作者：** 刘航天, 甘水滔, 张超, 张红旗, 孙文厚, 高子聪, 赵敏, 白雪  
**年份：** 2025  
**期刊：** 网络与信息安全学报  
**卷期：** 11(2): 26-49  

**摘要：**
随着物联网技术的广泛应用，物联网设备爆炸式增长。近年来，物联网设备导致的安全事件频发，使得物联网设备安全研究成为热点。首先，对物联网设备固件的安全特性（包括黑盒特性、网络特性和定制化特性）进行了深入剖析，这些特性给自动化漏洞挖掘带来了新的挑战。固件代码闭源、运行环境封闭、网络交互复杂、软硬件高度定制化，这些都增加了固件安全分析的难度。针对这些挑战，研究人员研发了一系列解决方案。通过对现有文献进行综合分析，从黑盒模糊测试、灰盒模糊测试、静态程序分析和固件重托管4个方面，总结了物联网设备固件自动化漏洞挖掘技术的最新研究进展。在此基础上，分析了尚存在的挑战和不足，并提出了未来的研究方向和思路，包括多种技术有机组合的发展趋势、大语言模型在自动化漏洞挖掘中的应用前景，以及物联网技术演进驱动漏洞挖掘技术的同步升级。

**研究内容：**
1. 物联网设备固件安全特性分析（黑盒特性、网络特性、定制化特性）
2. 固件分析面临的挑战（插桩困难、组件交互复杂、定制化导致仿真难度大）
3. 黑盒模糊测试技术综述（基于网络接口的测试、基于移动App的测试、基于LLM的测试）
4. 灰盒模糊测试技术综述（FIRM-AFL、SFuzz、EQUAFL、FirmHunter、FIRM-COV、EM-Fuzz、FirmFuzz等）
5. 静态程序分析技术（污点分析、符号执行）

**技术方案：**
1. 黑盒模糊测试方案
   - Labrador：基于代码覆盖率和距离测量的定向模糊测试
   - SmartTVFuzzer：基于日志引导的模糊测试
   - Snipuzz：基于消息片段的变异方法
   - VFuzz：基于Z-Wave协议的结构感知测试
   - SRFuzzer：基于键值对和配置模型的测试
   - ESRFuzzer：基于路由器的字典测试
   - ShadowBroker：基于中间人架构的MQTT测试
   - SIoTFuzzer：基于状态消息生成的测试
   - IoTScope：基于Web前端和状态分析发现隐藏接口
   - IoTFuzzer、Diane：基于移动App的测试
   - IoTBinding：建立状态机模型分析远程绑定安全性
   - LLMIF：利用大语言模型结合协议规范指导模糊测试

2. 灰盒模糊测试方案
   - Firm-AFL：高兼容性和高吞吐率的灰盒模糊测试
   - SFuzz：面向RTOS的灰盒模糊测试
   - EQUAFL：增强的用户模式模拟
   - FirmHunter：基于代码覆盖的模糊测试
   - FIRM-COV：协同调度和字典生成
   - EM-Fuzz：结合实时内存检查
   - FirmFuzz：静态分析辅助生成测试用例
   - NDFuzz：非侵入式代码覆盖率引导
   - BusyBox：增强的进程仿真
   - Li：利用硬件调试机制构建反馈驱动框架

3. 静态程序分析技术
   - DTaint、Andromeda、FlowDroid：污点分析
   - KLEE、SAGE、SPF、Pex、S2E：符号执行
   - 动态符号执行技术

**解决的问题：**
1. 物联网设备黑盒特性导致灰盒模糊测试难以应用
2. 设备插桩困难，无法构建有效的灰盒模糊测试模型
3. 设备组件交互复杂，静态程序分析追踪数据困难
4. 设备高度定制化，固件重托管模拟仿真难度大
5. 固件格式差异大，缺少统一标准
6. 符号信息缺失，影响静态分析准确性

**创新点：**
1. 系统性分析了物联网设备固件的三大安全特性及其挑战
2. 将大语言模型（LLM）引入物联网设备模糊测试领域
3. 提出了"黑盒-灰盒-静态分析-重托管"四种技术的融合框架
4. 强调多种技术有机组合的发展趋势
5. 指出物联网技术演进驱动漏洞挖掘技术同步升级的重要性

## 5. 物联网设备固件安全分析现状研究综述（2025年）

**论文标题：** 物联网设备固件安全分析现状研究综述  
**作者：** 马晓成, 王允超, 于洪洋, 魏强  
**年份：** 2025  
**期刊：** 信息安全学报  
**卷期：** 10(4)  

**摘要：**
随着物联网技术的发展及其应用场景的不断扩大，越来越多功能各异的物联网设备接入网络。万物互联时代到来的同时，大规模物联网设备暴露在互联网中，使得设备安全受到严峻的考验。固件是物联网设备中运行的软件，直接决定着设备的行为及功能，因此固件的安全性对保护物联网设备免受威胁至关重要。由于物联网技术分支众多且迭代迅速，针对物联网固件的安全测试研究也在不断的迭代更新。近年来，针对物联网固件的安全分析涌现了大量的研究，但缺乏专业的文献对研究现状进行及时的分析总结。本文首先梳理了物联网架构及当下设备的应用场景，然后介绍了固件的分类和通用工作框架，并讨论了固件可能面临的安全风险。基于物联网固件所面临的安全风险及其安全分析所面临的挑战，本文归纳出固件的一般流程，即总体分为固件获取、固件解包和文件提取、程序信息恢复和程序分析四个部分，并分别进行相关介绍。其中，程序分析部分是发现固件潜在问题必不可少的环节，本文从静态分析、动态环境构建及动态分析的角度进行介绍，对逆向分析、污点分析、符号执行、相似性检测、动态仿真环境构建及模糊测试等现有的固件程序分析技术进行详细分析和对比。最后总结当前工作的优势与不足，结合物联网固件的发展趋势，对未来的研究方向进行了展望。

**研究内容：**
1. 物联网架构及应用场景（网络设备、智能穿戴、智能家居、车联网、工业控制互联网、云端设备、边缘设备、应用程序、组件和服务层）
2. 固件分类（基于通用操作系统的固件、基于定制操作系统的固件、单片机固件）
3. 固件工作框架（应用软件层、组件和服务层、系统支撑层）
4. 固件面临的安全风险（Web管理端攻击、协议解析攻击、开源组件安全性、后门）
5. 固件分析一般流程（固件获取、固件解包和文件提取、程序信息恢复、程序分析）

**技术方案：**
1. 固件获取方法
   - 软件获取：官网公开下载、网络爬虫、模拟设备行为诱骗下载
   - 硬件获取：拆解设备、UART/JTAG接口转储、读取闪存芯片

2. 固件解包和文件提取
   - 针对文件系统的工具：Binwalk、BAT、firmware-mod-kit
   - 针对加密固件的工具：PkCrack、ARCHPR
   - 魔术值识别和替换
   - 自定义解包工具（Costin、Chen、Tien、Chen、Thomas、Redini等）

3. 程序信息恢复

## 总体对比与总结

### 年份分布

### 研究演进趋势
1. **从基础架构到具体技术**：2017年主要关注物联网架构层面的安全问题和宏观挑战；2021年开始系统性地从威胁、检测、防御三个维度进行技术综述
2. **从通用问题到固件安全**：2022年专门聚焦于固件这一物联网安全的核心载体；2025年两篇论文进一步深入到固件的自动化漏洞挖掘和安全分析技术细节
3. **技术手段的演进**：从传统的静态分析、符号执行发展到灰盒模糊测试、混合分析、机器学习辅助，最后到大语言模型赋能

### 核心技术分类
1. **固件获取与解包**：爬虫下载、硬件提取、App内提取、固件解包工具
2. **静态分析**：逆向分析、污点分析、符号执行、相似性检测
3. **动态分析**：黑盒模糊测试、灰盒模糊测试、仿真环境构建、覆盖率引导
4. **防御技术**：细粒度访问控制、安全协议设计、流量隐藏、可信计算、语音攻击防御

### 主要挑战
1. 固件代码闭源，难以获取和分析
2. 设备资源受限，难以部署传统安全措施
3. 硬件平台复杂异构，缺乏统一标准
4. 运行环境封闭，缺少调试接口
5. 物联网应用场景多样化，攻击面不断扩大

### 未来方向
1. **技术融合**：多种技术（静态+动态、灰盒+黑盒、AI+传统）有机组合
2. **智能化**：大语言模型、机器学习技术赋能漏洞挖掘
3. **自动化**：提升检测的自动化程度和效率
4. **精准化**：提高漏洞检测的准确率，减少误报
5. **扩展性**：构建通用框架，适应不同厂商和设备类型
