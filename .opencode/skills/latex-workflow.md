# LaTeX 论文编辑工作流

## 适用场景
当修改 .tex 文件后自动触发。

## 规则

### 编译
每次修改 .tex 文件后，**必须**立即执行编译命令验证修改结果，不要等用户提醒：
```
latexmk -pdf -xelatex -interaction=nonstopmode main.tex
```

### 编译时机
- 修改任何 .tex 文件内容后立即编译
- 多次连续修改时，在最后一次修改完成后编译一次即可
- 如果用户明确说"重新编译"，立即执行编译

### 编译后
- 检查编译输出是否有错误
- 如果有错误，分析原因并修复后重新编译
- 只需简要报告编译结果（成功/失败），不要展示完整编译日志

### 提交
编译成功后，**必须**立即执行 git commit 提交修改：
- 自动生成简洁的中文提交信息，概括本次修改内容
- 只提交 .tex 相关文件，不要提交 .aux、.log、.pdf 等编译产物
- 不要 push 到远程，除非用户明确要求
