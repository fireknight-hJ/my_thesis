# 3.3 驱动函数分析、替换与编译模块

本章详细阐述驱动函数分析、替换与编译模块的设计与实现。

## 3.3.1 模块概述

本模块是系统的核心智能层，负责基于静态分析结果对驱动函数进行语义理解和代码替换。模块采用大语言模型（LLM）驱动的智能体架构，通过工具调用和工作流编排实现自动化的函数分析与替换。

## 3.3.2 智能体工作流设计

### 3.3.2.1 LangGraph工作流框架

系统采用LangGraph框架构建图式工作流，核心包含三个逻辑节点：

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  推理与决策  │ ──→ │  工具调用   │ ──→ │  结果汇总   │
│    节点     │ ←── │    节点     │     │    节点     │
└─────────────┘     └─────────────┘     └─────────────┘
```

**图 3-2 智能体工作流架构图**

节点职责说明：

| 节点 | 职责 | 输入 | 输出 |
|------|------|------|------|
| 推理与决策 | 判断函数类别、选择替换策略 | 函数源码、上下文 | 工具调用请求或最终答案 |
| 工具调用 | 执行CodeQL查询、获取函数信息 | 工具名和参数 | 工具执行结果 |
| 结果汇总 | 结构化整理输出并写入缓存 | 对话历史 | FunctionClassifyResponse |

### 3.3.2.2 状态机设计

```python
class AgentState(MessagesState):
    messages: List[Message]
    final_response: FunctionClassifyResponse
    function_name: str
```

状态转移规则：
1. `START` → `agent`：初始化状态，开始处理
2. `agent` → `tools`：如果LLM请求工具调用
3. `agent` → `respond`：如果LLM给出最终答案
4. `tools` → `agent`：工具执行完毕，返回推理
5. `respond` → `END`：输出最终结果

## 3.3.4 替换策略体系

### 3.3.4.1 替换策略分类

本研究总结出四种通用替换方式：

**策略1：跳过硬件副作用（SKIP）**

适用于：初始化类、控制类函数

```c
// 原始代码
void HAL_UART_Enable(UART_HandleTypeDef *huart) {
    huart->Instance->CR1 |= USART_CR1_UE;
}

// 替换代码
void HAL_UART_Enable(UART_HandleTypeDef *huart) {
    (void)huart;
}
```

**策略2：剥离硬件代码、保留控制逻辑（STRIP）**

适用于：发送类函数

```c
// 原始代码
HAL_StatusTypeDef HAL_UART_Transmit(UART_HandleTypeDef *huart, 
                                     uint8_t *pData, uint16_t Size) {
    huart->gState = HAL_UART_STATE_BUSY_TX;
    for (int i = 0; i < Size; i++) {
        huart->Instance->DR = pData[i];
        while (!(huart->Instance->SR & USART_SR_TXE));
    }
    huart->gState = HAL_UART_STATE_READY;
    return HAL_OK;
}

// 替换代码
HAL_StatusTypeDef HAL_UART_Transmit(UART_HandleTypeDef *huart, 
                                     uint8_t *pData, uint16_t Size) {
    huart->gState = HAL_UART_STATE_BUSY_TX;
    lcmhal_uart_transmit(huart->Instance, pData, Size);
    huart->gState = HAL_UART_STATE_READY;
    return HAL_OK;
}
```

**策略3：替换为仿真I/O（REDIRECT）**

适用于：接收/发送类函数

```c
HAL_StatusTypeDef HAL_UART_Receive(UART_HandleTypeDef *huart, 
                                    uint8_t *pData, uint16_t Size, 
                                    uint32_t Timeout) {
    return lcmhal_uart_receive(huart->Instance, pData, Size, Timeout);
}
```

**策略4：事件驱动模拟（EVENT）**

适用于：中断回调类函数

```c
void HAL_UART_RxCpltCallback(UART_HandleTypeDef *huart) {
    lcmhal_event_t event = {
        .type = LCMHAL_EVENT_UART_RX_COMPLETE,
        .handle = huart
    };
    lcmhal_event_post(&event);
    process_received_data(huart->pRxBuffPtr);
}
```

## 3.3.5 替换验证机制

### 3.3.5.1 语法正确性检查

```python
def verify_replacement(func_name: str, replace_code: str) -> dict:
    result = {"pass": False, "errors": [], "warnings": []}
    
    compile_result = compile_check(replace_code)
    if compile_result.errors:
        result["errors"].extend(compile_result.errors)
        return result
    
    if not check_signature_match(func_name, replace_code):
        result["errors"].append("Function signature mismatch")
        return result
    
    result["pass"] = True
    return result
```

### 3.3.5.2 替换规则检查

```python
def check_replacement_validation(original_code: str, replace_code: str) -> dict:
    checks = [
        ("函数名一致", check_function_name_consistency),
        ("参数列表一致", check_parameter_consistency),
        ("返回类型一致", check_return_type_consistency),
        ("无未声明变量", check_undeclared_variables),
        ("无无限循环", check_infinite_loops),
        ("保留关键状态更新", check_state_preservation),
    ]
    
    results = []
    for check_name, check_func in checks:
        passed, message = check_func(original_code, replace_code)
        results.append((check_name, passed, message))
    
    return {"pass": all(r[1] for r in results), "results": results}
```
