# 3.2 驱动函数接口静态分析与识别模块

本章详细阐述驱动函数接口静态分析与识别模块的设计与实现。

## 3.2.1 模块概述

本模块是整个方案的基础支撑层，负责自动、精准地定位固件中与硬件交互的代码，并将其抽象为标准化的可查询知识库。模块基于CodeQL静态分析引擎实现，通过预定义的查询规则从固件源码中提取驱动相关信息。

## 3.2.2 CodeQL数据库构建

**构建流程**包含以下步骤：

1. **源码准备**：将固件源代码组织为标准项目结构
2. **数据库创建**：调用`codeql database create`命令创建CodeQL数据库
3. **索引建立**：对C/C++源文件建立语法树、控制流、数据流索引

```bash
codeql database create <db_path> \
    --language=cpp \
    --source-root=<src_path> \
    --overwrite
```

数据库核心组件包括：
- 抽象语法树（AST）：源码的层次化树状表示
- 控制流图（CFG）：函数内语句执行顺序
- 数据流图（DFG）：数据在程序中的传播路径
- 符号表：变量、函数、类型的定义信息

## 3.2.3 MMIO访问模式识别

### 3.2.3.1 MMIO结构体定位

本研究将MMIO访问识别分为两步：定位MMIO结构体实例与捕获字段访问。

典型MMIO结构体定义（以STM32 HAL库为例）：

```c
typedef struct {
  __IO uint32_t SR;    /*!< Status register */
  __IO uint32_t DR;    /*!< Data register */
  __IO uint32_t BRR;   /*!< Baud rate register */
  __IO uint32_t CR1;   /*!< Control register 1 */
  __IO uint32_t CR2;   /*!< Control register 2 */
} USART_TypeDef;
```

定位MMIO结构体实例的三种典型模式：

| 模式 | 描述 | 示例代码 |
|------|------|----------|
| 直接地址强转 | 将固定地址强制转换为外设结构体指针 | `(USART_TypeDef *)0x40011000` |
| 间接初始化 | 通过配置表将外设基地址赋值给句柄 | `huart->Instance = USART1` |
| 常量指针参数 | 函数以结构体指针形式接收外设基址 | `HAL_UART_Transmit(USART_TypeDef *huart, ...)` |

### 3.2.3.2 CodeQL查询实现

```ql
import cpp

from Variable v, Type t
where
  t.getName().regexpMatch(".*_TypeDef") and
  v.getType() = t
select v, v.getLocation().getFile().getBaseName() as file,
       v.getLocation().getStartLine() as line
```

MMIO字段访问捕获查询：

```ql
import cpp

from FieldAccess fa, Field f
where
  f = fa.getTarget() and
  f.getName().regexpMatch("DR|SR|CR[0-9]*|ISR")
select fa, fa.getEnclosingFunction().getName() as func,
       f.getName() as field, fa.getLocation().getStartLine() as line
```

## 3.2.4 函数分类体系

### 3.2.4.1 硬件相关函数分类

在函数粒度上区分两类与硬件相关的函数：

- **MMIO函数**：函数内部存在直接的寄存器字段访问
- **驱动函数**：函数通过驱动句柄间接参与硬件交互

### 3.2.4.2 驱动函数语义分类

| 类别 | 缩写 | 描述 | 典型示例 |
|------|------|------|----------|
| 接收类 | RECV | 从硬件读取数据 | `HAL_UART_Receive` |
| 发送类 | SEND | 向硬件写入数据 | `HAL_UART_Transmit` |
| 中断类 | IRQ | 中断处理函数 | `HAL_UART_IRQHandler` |
| 初始化类 | INIT | 外设初始化 | `HAL_UART_Init` |
| 控制类 | CTRL | 启停、模式切换 | `HAL_UART_Start` |
| 轮询类 | POLL | 轮询状态位 | `__HAL_UART_GET_FLAG` |
| DMA类 | DMA | 配置DMA传输 | `HAL_UART_Transmit_DMA` |
| 非驱动 | NODRIVER | 纯软件函数 | 业务逻辑函数 |

自动分类CodeQL查询：

```ql
import cpp

from Function f, int mmioCount
where
  mmioCount = count(FieldAccess fa | 
    fa.getEnclosingFunction() = f and
    fa.getTarget().getName().regexpMatch("DR|SR|CR[0-9]*|ISR")
  )
select f.getName(), mmioCount,
       case 
         when f.getName().regexpMatch(".*_Receive") then "RECV"
         when f.getName().regexpMatch(".*_Transmit") then "SEND"
         when f.getName().regexpMatch(".*_IRQ") then "IRQ"
         when f.getName().regexpMatch(".*_Init") then "INIT"
         when mmioCount > 0 then "MMIO"
         else "NODRIVER"
       end as category
```

## 3.2.5 数据流追踪

以MMIO字段访问为源点，以函数参数为汇点，构建数据流配置：

```ql
import cpp
import semmle.code.cpp.dataflow.TaintTracking

class MMIODataFlow extends TaintTracking::Configuration {
  MMIODataFlow() { this = "MMIODataFlow" }

  override predicate isSource(DataFlow::Node source) {
    exists(FieldAccess fa |
      fa.getTarget().getName().regexpMatch("DR|SR|CR[0-9]*") and
      source.asExpr() = fa
    )
  }

  override predicate isSink(DataFlow::Node sink) {
    sink instanceof ParameterNode
  }
}
```

本节详细阐述了驱动函数接口静态分析与识别模块的设计与实现。该模块基于CodeQL静态分析引擎，通过MMIO结构体定位、字段访问捕获、函数分类和数据流追踪等核心功能，实现了对固件中硬件相关代码的自动识别，并以统一的数据模型输出分析结果，为后续的驱动替换提供了精确的输入数据。
